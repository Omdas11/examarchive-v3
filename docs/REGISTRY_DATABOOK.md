## Registry Schema

Each entry in the registry (`SyllabusRegistryEntry`) contains:

| Field | Type | Description |
|---|---|---|
| `paper_code` | string | Unique paper identifier (e.g. `PHYDSC101T`) |
| `paper_name` | string | Full descriptive name of the paper |
| `semester` | number | Semester number (1–8) |
| `subject` | string | Subject / disciplinary area |
| `credits` | number | Credit weighting |
| `programme` | string | Academic framework (`FYUGP`, `CBCS`) |
| `university` | string | University name |
| `category` | string | Paper category (DSC, DSM, SEC, IDC, etc.) |
| `contact_hours` | number? | Total contact hours |
| `full_marks` | number? | Maximum marks |
| `course_objective` | string? | Course objective / description |
| `learning_outcomes` | string? | Expected learning outcomes |
| `units` | SyllabusUnit[]? | Structured unit breakdown with topics |
| `reference_books` | string[]? | Recommended reading list |

## Helper Functions

The following TypeScript helper functions are exported from `src/data/syllabus-registry.ts`:

| Function | Signature | Description |
|---|---|---|
| `findByPaperCode` | `(code, university?) → entry?` | Look up one entry by code |
| `getByUniversity` | `(univ, programme?, category?) → entries` | Filter by university |
| `getBySubject` | `(subject, university?, programme?) → entries` | Filter by subject |
| `groupBySemester` | `(entries) → Map<number, entries>` | Group entries by semester |
| `getAllUniversities` | `() → string[]` | All unique universities |
| `getAllSubjects` | `(university?, programme?) → string[]` | All unique subjects |

## Adding New Entries

1. Open `src/data/syllabus-registry.ts`
2. Append objects conforming to `SyllabusRegistryEntry` in the appropriate section
3. Commit and push — the website will auto-render new entries in the Syllabus Library
4. Update this data book to reflect the addition
